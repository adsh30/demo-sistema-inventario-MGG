● Mineral Group Guayana C.A.

Gestión de inventarios a la medida de la operación minera.

Demo del nuevo ecosistema digital modular para Mineral Group Guayana. Una solución creada desde cero que evoluciona junto a la operación, sustituyendo las herramientas manuales y genéricas por procesos automatizados y adaptados al flujo real del sector.

Entrar al sistema Ver fases del proyecto
Fase 1 · Módulos disponibles

Un monolito modular, listo para crecer

Cada módulo se compone de repository, view y un module de entrada; el de órdenes añade controller con las reglas de negocio. El kernel shared/ agrupa store, auth, router, UI y notificaciones.

▦

Dashboard

KPIs en tiempo real: stock crítico, órdenes pendientes, facturación del mes y actividad reciente.

⬢

Inventario

Productos con SKU, categoría, stock mínimo y alertas. Preparado para multi-almacén en Fase 2.

⚒

Proveedores

Registro con RIF, contacto, categorías y estado. Base del mercado de proveedores para licitaciones.

✉

Órdenes de pedido

La analista crea, la administradora aprueba o rechaza. Cambio de proveedor, desistimientos y cancelaciones con historial.

$

Facturación

Genera facturas desde órdenes aprobadas. Estado pendiente / pagada / anulada.

◐

Roles y permisos

Administradora y analista, con vistas y acciones diferenciadas en cada módulo.

Plan de entrega

Tres fases, foco quirúrgico

Entregamos valor temprano sin sobre-construir. Fase 1 reemplaza el Excel; Fase 2 cubre el flujo completo de compra/venta y licitaciones; Fase 3 cierra brechas operativas.

FASE 01 · EN ENTREGA

Núcleo operativo

  • Landing pública + login con roles
  • Dashboard ejecutivo
  • Inventario (productos, stock, alertas)
  • Proveedores (registro, categorías)
  • Órdenes (crear, aprobar, rechazar, cancelar, cambiar proveedor, desistimiento)
  • Facturación con IVA
FASE 02

Compra-Venta, licitación y producción

  • Módulo de Compra
  • Módulo de Venta
  • Notificación de licitación por rol
  • Almacén general + almacenes por departamento
  • Transferencias internas entre almacenes
  • Flujo "primero en tomar la orden gana"
  • Producción / Procesos: hornos y otras operaciones que consumen insumos (BOM, salidas por consumo, trazabilidad)
FASE 03

Operación y analítica

  • Reportes y exportes (PDF / Excel)
  • Bitácora de auditoría
  • KPIs e indicadores históricos
  • Plus: Gestión de combustible para la flota vehicular
Tecnología

Stack del demo y del producto

Esta demo está en HTML/CSS/JS puro para que corra en GitHub Pages y se valide rápido. La arquitectura modular se conserva al migrar.

◇

Demo actual

HTML5 · CSS3 · JavaScript modular · LocalStorage · npm run dev con serve.

◈

Producto objetivo

React + Tailwind · Next.js / Node · Supabase (Postgres + Auth + Storage) · DigitalOcean.

⌥

Flujo de trabajo

GitHub con Gitflow mínimo: main / develop / feature/*. CI básico y previews por PR.
